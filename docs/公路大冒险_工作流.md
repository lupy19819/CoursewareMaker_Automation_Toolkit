# 公路大冒险专项生成与校验规则

> 整理自 2026-05-12 实战，生成策略为复制替换型。
> 从基准配置 deepcopy 关卡骨架，替换可变内容（题干文本/音频、选项图片、正确答案、小鹿动效）。
> 意图判定、素材同名确认、任务清单、新建游戏、导入保存、回读比对、预览和发布都由主流程 `workflow/` 控制。

---

## 一、主流程挂载点

```text
题目数据 JSON（题干文本/音频、选项三态图、正确项）
    ↓ python3 scripts/generate_guoji_l2_shu4_road.py --input <题目.json> --template <参考配置.json> --output <输出.config.json>
输出配置 JSON + build-meta
    ↓ 后续导入保存、回读、预览、发布由主流程执行
```

## 二、生成脚本

**路径**：`scripts/generate_guoji_l2_shu4_road.py`  
**基准配置**：`reference_configs/road_adventure_ref.json`（国际level1公路大冒险暑4，绿地模板，已纳入 git 固定参考）

### 运行

```bash
cd CoursewareMaker_Automation_Toolkit
python3 scripts/generate_guoji_l2_shu4_road.py
```

输出：`output/国际level2公路大冒险暑4_config.json`

正式任务必须使用动态输入：

```bash
python3 scripts/generate_guoji_l2_shu4_road.py \
  --input data/road_adventure_questions.json \
  --template reference_configs/road_adventure_ref.json \
  --output output/road_adventure_configs/<游戏名>.json \
  --meta output/road_adventure_configs/<游戏名>.build-meta.json
```

## 三、题目数据格式

```json
{
  "questions": [
    {
      "stem_text": "Carry the shopping inside.",
      "audio_url": "https://.../stem.mp3",
      "options": [
        {"image_urls": {"m": "https://.../a_m.png", "d": "https://.../a_d.png", "c": "https://.../a_c.png"}, "correct": false},
        {"image_urls": {"m": "https://.../b_m.png", "d": "https://.../b_d.png", "c": "https://.../b_c.png"}, "correct": true},
        {"image_urls": {"m": "https://.../c_m.png", "d": "https://.../c_d.png", "c": "https://.../c_c.png"}, "correct": false}
      ]
    }
  ]
}
```

| 字段 | 说明 |
|------|------|
| `stem_text` | 题干文本；纯文本和音频+文本题必填 |
| `audio_url` / `audio_key` | 题干音频；纯音频和音频+文本题必填 |
| `image_urls.m/d/c` | 选项三态图片 URL，题目相关，必须来自输入或素材确认结果 |
| `correct` | 是否为正确选项，每关必须且只能有 1 个 |

题干形态：

| 类型 | 输入要求 | 写入规则 |
|---|---|---|
| `pure_text` | `stem_text` | 写入参考模板中明确的题干 MLabel；清空题干音频，避免继承旧音频 |
| `pure_audio` | `audio_url` 或 `audio_key` | 写入语音组件 `MAudio.value` |
| `audio_text` | `stem_text` + `audio_url/audio_key` | 同时写题干文本和题干音频 |

如果输入了 `stem_text`，但参考模板没有明确的题干文本 MLabel 承载节点，生成脚本必须阻断，不能把文字猜写到选项或装饰组件。

脚本内 `IMAGES/AUDIOS/QUESTIONS` 只保留为旧样例兼容；正式任务不得依赖修改脚本常量。背景、模板骨架、小鹿动效等固定模板资源由 `--template` 参考配置继承。

## 四、组件渲染顺序

**⚠️ 渲染层级 = `components[]` 数组顺序（越靠后越在上层），zIndex 字段仅作编辑器排序参考，不影响运行时渲染。**

公路大冒险为「复制替换型」游戏，**所有组件顺序来自基准配置 deepcopy，不在生成脚本中重排**。只要基准模板的 `components[]` 顺序正确，生成结果即正确。

标准层级顺序（从底层→顶层，即数组从前→后）：

```
[背景(绿地/沙漠)] → [装饰层] → [语音按钮]
→ [选项节点(节点/节点_103/节点_104, 含图片×3态)] → [AloneClickChoice(点击判定)]
→ [小鹿开车(Spine, 含正确/一般/错误动效)]
```

---

## 五、生成策略

**复制替换型**（非动态布局）：

1. 从基准配置 deepcopy 全部关卡
2. 逐关替换可变内容：
   - 题干文本：写入参考模板中明确的题干 MLabel
   - 语音按钮音频：`语音` 组件「播放语音」state → `MAudio.value`；纯文本题清空音频
   - 选项图片三态：`AloneClickChoice` 配对「节点」→ 默认/当前组件正确/当前组件错误 → `MSprite.value`
   - 正确答案标记：`AloneClickChoice.anwserConfig.anwserRadio` → 1(正确) / 2(错误)
   - 小鹿开车正确动效：根据正确选项位置 → `MSpine.animation`（xia/zhong/shang）
3. **不重算坐标** — 所有组件位置、尺寸沿用基准配置原值

### 正确项动效映射

| 正确选项位置 | 小鹿动效 animation | 备注 |
|------------|-------------------|------|
| 选项1（上方） | `xia` | 动效名与位置名相反（底部=xia） |
| 选项2（中间） | `zhong` | |
| 选项3（下方） | `shang` | 动效名与位置名相反（顶部=shang） |

## 六、选项图片命名规则

每个选项有三张匹配图，命名后缀对应三态：

```
{key}_m.png  → 默认态
{key}_d.png  → 当前组件正确
{key}_c.png  → 当前组件错误
```

## 八、模板类型

公路大冒险有两种视觉模板：

| 模板 | 说明 | 互换性 |
|------|------|--------|
| 绿地模板 | 绿色草地背景 | 可互换（仅背景图/MSprite URL 不同） |
| 沙漠模板 | 沙漠背景 | 可互换 |

> 生成脚本使用绿地模板为基准，如需沙漠模板，替换基准配置路径即可。

## 九、调试备忘

| 问题 | 原因 | 解法 |
|------|------|------|
| 选项图片不切换 | 三态图片 URL 中 `_m/_d/_c` 后缀不完整 | 检查 `IMAGES` 字典中每选项的三个 key |
| 正确项动效不对 | `anwserRadio` 设置错误 | 确认正确选项索引对应的 `anwserRadio` = 1 |
| 语音不播放 | 音频 URL 失效 | 确认 `AUDIOS` 字典中音频 URL 可访问 |
| 选项数不对 | 基准配置选项数 ≠ 目标选项数 | 选取匹配选项数的基准配置 |
