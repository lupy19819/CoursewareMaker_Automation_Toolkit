# 过桥大冒险 动态生成工作流

> 整理自 2026-05-11 实战，完全替代原手动配置流程。
> 核心思路：从题目数据动态生成 JSON，通过 Python HTTP API 直接保存到 CoursewareMaker。

---

## 一、整体流程

```
题目数据（LEVELS 常量）
    ↓ python3 scripts/generate_guoqiao_shu2.py
output/guoqiao_shu2/guoqiao_shu2.json
    ↓ python3（urllib.request PUT API）
CoursewareMaker 平台保存
    ↓ createPreviewUrl API
预览链接（有效期7天）
```

## 三、题目数据格式（LEVELS）

每个关卡定义如下：

```python
{
    'parts': [
        {'type': 'fixed', 'text': 'Kuan'},        # 固定词（桥墩）
        {'type': 'drag',  'text': 'enjoys'},       # 拖拽槽（答案）
        {'type': 'fixed', 'text': 'riding'},
        {'type': 'fixed', 'text': 'bikes.'},       # 末尾带句号（由句号组件显示，桥墩标签会自动去掉句点）
    ],
    'options': ['enjoys', 'like'],                 # 拖拽选项（含正确项+干扰项）
    'quiz_img': 'https://...cos.../xxx.png',       # 题目配图 URL
    'audio': '',                                   # 音频 URL，无音频留空
}
```

**注意**：
- `parts` 中末尾固定词可带 `.`，脚本会自动从显示标签中去掉句点（由独立句号组件承担）
- `options` 里的选项显示也自动去掉末尾句点
- `audio` 为空时，不生成喇叭和点击选择组件

## 四、组件渲染顺序

**⚠️ 渲染层级 = `components[]` 数组顺序（越靠后越在上层），zIndex 字段仅作编辑器排序参考，不影响运行时渲染。**

组装组件时必须严格按以下顺序：
```
[背景] → [题目配图] → [关卡数组件] → [喇叭低层级] → [小鹿开车]
→ [桥面组件(slot+pillar混排)] → [砖块掉落] → [句号] → [节点]
→ [选项]  ← 最后 = 最顶层，保证可拖拽交互
```

## 五、布局规则

### 5.1 桥区横向填满

```python
BRIDGE_LEFT  = -643   # 桥区左边界（组件左边缘）
BRIDGE_RIGHT =  548   # 桥区右边界（组件右边缘）
BRIDGE_TOTAL = 1191   # 总宽
GAP_BRIDGE   = 0      # 桥墩/槽之间紧贴，无间距
```

所有桥面组件（桥墩+槽）按比例等比缩放后填满 [-643, +548]。

### 4.2 统一槽宽（防止提示）

同一关卡内，**所有拖拽放置区和选项宽度相同**，取该关最宽 drag part 的自然宽度作为基准，等比缩放后统一使用。干扰项同等宽度。

### 4.3 选项间距自适应

```python
OPTION_AREA_W = 1200  # 选项可用总宽

# 选项间距公式
gap = max(16, min(100, (OPTION_AREA_W - sum(opt_widths)) / (n - 1)))
```

### 4.4 题目配图位置

| 情况 | x 坐标 | y 坐标 |
|------|--------|--------|
| 有音频 | 91.49（偏右，给喇叭留位置） | 355 |
| 无音频 | 0（居中） | 355 |

图片固定尺寸：**447 × 297**

### 4.5 自适应字号

```python
FONT_DEFAULT  = 51
FONT_MIN      = 35
CHAR_W_FACTOR = 0.55   # 英文粗体字符宽度系数

# 公式
max_chars = max(len(t) for t in all_texts_in_level)
font_size = clamp(int(unified_slot_w / (max_chars * 0.55)), FONT_MIN, FONT_DEFAULT)
```

字号应用于该关卡内所有桥墩标签和选项标签。

### 4.6 无音频关卡

自动省略：
- 喇叭低层级
- 点击选择

## 六、保存到平台（Python API）

**无需浏览器 CDP**，直接用 Python `urllib.request` 调用：

```python
import json, urllib.request

GAME_ID = '<目标游戏 game_id>'
TOKEN   = '<localStorage["GAMEMAKER_TOKEN"]>'
COOKIE  = 'oneLoginToken=<cookie>'  # 从浏览器 DevTools 复制

# 1. GET 游戏元数据
req = urllib.request.Request(
    f'https://sszt-gateway.speiyou.com/beibo/game/config/game?game_id={GAME_ID}',
    headers={'beibotoken': TOKEN, 'Cookie': COOKIE}
)
with urllib.request.urlopen(req) as resp:
    meta = json.loads(resp.read())['result']

# 2. 读取生成的配置
with open('output/guoqiao_shu2/guoqiao_shu2.json') as f:
    cfg = json.load(f)

# 3. 拼装 payload（与手动保存保持一致：保留完整元信息，只替换 configuration）
payload = dict(meta)
payload['game_id'] = GAME_ID
payload['components'] = payload.get('components') if isinstance(payload.get('components'), list) else []
payload['configuration'] = cfg  # ⚠️ 必须是 dict，不是字符串

# 4. PUT 保存
body = json.dumps(payload, ensure_ascii=False).encode('utf-8')
req2 = urllib.request.Request(
    'https://sszt-gateway.speiyou.com/beibo/game/config/game',
    data=body,
    headers={'beibotoken': TOKEN, 'Cookie': COOKIE,
             'Content-Type': 'application/json; charset=utf-8'},
    method='PUT'
)
with urllib.request.urlopen(req2) as resp2:
    print(json.loads(resp2.read()))   # {'code': 0, 'result': None, 'msg': 'success'}

# 5. 生成预览链接（有效期7天）
body3 = json.dumps({
    'game_id': GAME_ID,
    'base_preview_url': 'https://coursewaremaker.speiyou.com/#/share-preview'
}).encode()
req3 = urllib.request.Request(
    'https://sszt-gateway.speiyou.com/beibo/game/config/createPreviewUrl',
    data=body3,
    headers={'beibotoken': TOKEN, 'Cookie': COOKIE, 'Content-Type': 'application/json'},
    method='POST'
)
with urllib.request.urlopen(req3) as resp3:
    r = json.loads(resp3.read())
    print(r['result']['preview_url'])
```

**关键注意**：
- `configuration` 字段传 **dict 对象**，不能是 JSON 字符串（否则会双重编码，引擎崩溃）
- `components`（组件库大字段约32MB）必须排除，否则超出服务端 16MB 限制
- Token 存于浏览器 `localStorage["GAMEMAKER_TOKEN"]`，可用 DevTools 获取
- Cookie 用 `oneLoginToken=...`，从 DevTools Application → Cookies 复制

## 七、顶层 JSON 必要字段

引擎 `analysisComponentsJson` 启动时会读取以下字段，**缺一不可**：

```
{
  "meta":       { ...从参考配置 deepcopy... },
  "additional": { ...从参考配置 deepcopy... },
  "components": [ ...组件类型注册表（9条），从参考配置 deepcopy... ],
  "common":     { ...公共层，deepcopy 参考配置并替换 game 层不用的部分... },
  "game":       [ ...各关卡... ]
}
```

生成脚本已自动从参考配置 deepcopy 以上字段，无需手动处理。

## 十一、调试备忘

| 问题 | 原因 | 解法 |
|------|------|------|
| `Cannot read properties of undefined (reading 'length')` | 顶层缺 `components` 注册表 | 从参考配置 deepcopy `meta/additional/components` |
| `setResUrlFromArr` 崩溃 | 生成的关卡结构缺少 `levelData.uiConfig` | 确认 level 骨架来自参考配置 deepcopy |
| PUT 返回 404 | 用了错误的 API 路径或 method | 确认是 `PUT /beibo/game/config/game`（不是 POST /save） |
| Token 过期 | JWT exp 过期 | 重新从浏览器 localStorage 获取 `GAMEMAKER_TOKEN` |
| WARN: 桥墩3 MSprite | 3槽模板跨组件复用正常 | 可忽略 |
