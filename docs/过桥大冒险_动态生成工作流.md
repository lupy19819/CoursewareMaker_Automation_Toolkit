# 过桥大冒险专项生成与校验规则

> 整理自 2026-05-11 实战，完全替代原手动配置流程。
> 核心思路：从题目数据动态生成 JSON。意图判定、素材同名确认、任务清单、新建游戏、导入保存、回读比对、预览和发布都由主流程 `workflow/` 控制。

---

## 一、主流程挂载点

```text
题目数据 JSON（句子、桥面结构、选项、题图、音频）
    ↓ python3 scripts/generate_guoqiao_shu2.py --input <题目.json> --template <参考配置.json> --output <输出.config.json>
输出配置 JSON + build-meta
    ↓ 后续导入保存、回读、预览、发布由主流程执行
```

## 三、题目数据格式

每个关卡定义如下：

```json
{
  "levels": [
    {
      "sentence": "Kuan enjoys riding bikes.",
      "parts": [
        {"type": "fixed", "text": "Kuan"},
        {"type": "drag", "text": "enjoys"},
        {"type": "fixed", "text": "riding"},
        {"type": "fixed", "text": "bikes."}
      ],
      "options": ["enjoy", "enjoys"],
      "quiz_img": "https://.../quiz.png",
      "audio": "https://.../stem.mp3"
    }
  ]
}
```

**注意**：
- `parts` 中末尾固定词可带 `.`，脚本会自动从显示标签中去掉句点（由独立句号组件承担）
- `options` 里的选项显示也自动去掉末尾句点
- `audio` 是题目音频，正式任务必须来自输入或素材确认结果；旧样例允许为空，但正式生成会阻断空音频
- 脚本内 `LEVELS/IMG_BASE` 只保留为旧样例兼容；正式任务不得依赖修改脚本常量
- 参考配置中的背景、桥墩皮肤、砖块、小鹿、公共层等固定模板资源可直接继承

## 四、组件渲染顺序

**⚠️ 渲染层级 = `components[]` 数组顺序（越靠后越在上层），zIndex 字段仅作编辑器排序参考，不影响运行时渲染。**

组装组件时必须严格按以下顺序：
```
[背景] → [题目配图] → [关卡数组件] → [喇叭低层级] → [小鹿开车]
→ [桥面组件(slot+pillar混排)] → [砖块掉落] → [句号] → [节点]
→ [选项]  ← 最后 = 最顶层，保证可拖拽交互
```

## 五、布局规则

### 5.1 桥区横向填满

```python
BRIDGE_LEFT  = -643   # 桥区左边界（组件左边缘）
BRIDGE_RIGHT =  548   # 桥区右边界（组件右边缘）
BRIDGE_TOTAL = 1191   # 总宽
GAP_BRIDGE   = 0      # 桥墩/槽之间紧贴，无间距
```

所有桥面组件（桥墩+槽）按比例等比缩放后填满 [-643, +548]。

### 4.2 统一槽宽（防止提示）

同一关卡内，**所有拖拽放置区和选项宽度相同**，取该关最宽 drag part 的自然宽度作为基准，等比缩放后统一使用。干扰项同等宽度。

### 4.3 选项间距自适应

```python
OPTION_AREA_W = 1200  # 选项可用总宽

# 选项间距公式
gap = max(16, min(100, (OPTION_AREA_W - sum(opt_widths)) / (n - 1)))
```

### 4.4 题目配图位置

| 情况 | x 坐标 | y 坐标 |
|------|--------|--------|
| 有音频 | 91.49（偏右，给喇叭留位置） | 355 |
| 无音频 | 0（居中） | 355 |

图片固定尺寸：**447 × 297**

### 4.5 自适应字号

```python
FONT_DEFAULT  = 51
FONT_MIN      = 35
CHAR_W_FACTOR = 0.55   # 英文粗体字符宽度系数

# 公式
max_chars = max(len(t) for t in all_texts_in_level)
font_size = clamp(int(unified_slot_w / (max_chars * 0.55)), FONT_MIN, FONT_DEFAULT)
```

字号应用于该关卡内所有桥墩标签和选项标签。

### 4.6 无音频关卡

自动省略：
- 喇叭低层级
- 点击选择

## 六、主流程保存边界

本文只产出配置 JSON。保存到已有 `game_id`、新建游戏、回读比对、预览和发布必须回到主流程，由 `workflow_planner.py` 和 `workflow_guard.py` 决定是否允许执行。

保存语义统一由主流程脚本负责：`GET 完整游戏元信息 -> 只替换 configuration -> 保留或补齐 components=[] -> PUT /beibo/game/config/game`。`configuration` 必须是对象，不能双重编码成字符串。

---

## 七、顶层 JSON 必要字段

引擎 `analysisComponentsJson` 启动时会读取以下字段，**缺一不可**：

```
{
  "meta":       { ...从参考配置 deepcopy... },
  "additional": { ...从参考配置 deepcopy... },
  "components": [ ...组件类型注册表（9条），从参考配置 deepcopy... ],
  "common":     { ...公共层，deepcopy 参考配置并替换 game 层不用的部分... },
  "game":       [ ...各关卡... ]
}
```

生成脚本已自动从参考配置 deepcopy 以上字段，无需手动处理。

## 十一、调试备忘

| 问题 | 原因 | 解法 |
|------|------|------|
| `Cannot read properties of undefined (reading 'length')` | 顶层缺 `components` 注册表 | 从参考配置 deepcopy `meta/additional/components` |
| `setResUrlFromArr` 崩溃 | 生成的关卡结构缺少 `levelData.uiConfig` | 确认 level 骨架来自参考配置 deepcopy |
| PUT 返回 404 | 用了错误的 API 路径或 method | 确认是 `PUT /beibo/game/config/game`（不是 POST /save） |
| Token 过期 | JWT exp 过期 | 重新从浏览器 localStorage 获取 `GAMEMAKER_TOKEN` |
| WARN: 桥墩3 MSprite | 3槽模板跨组件复用正常 | 可忽略 |
