> 整理自 2026-04-30 调试经历，用于避免生成配置时缺字段导致编辑器报错。

## 顶层 JSON 结构

```
{
  "components": [ ... ],   // 组件注册表（全局）
  "game": [                // 关卡数组，每关一个对象
    {
      "components": [ ... ]  // 该关卡所有组件列表
    }
  ]
}
```

## 关卡组件列表（game[i].components[]）

### ⚠️ 必须字段（顶层，与 component_data 同级）

```json
{
  "component_id": "...",       // ❌ 不能为空字符串！见下表
  "component_name": "...",     // 显示名
  "component_url": "...",      // bundle URL（BaseComponent 为空）
  "name": "...",               // 类名（如 LDragPlace）
  "version": "...",            // 版本（如 0.2.0）
  "index": 0,
  "component_data": { ... }
```

## component_data 内部结构

### 必须字段

```json
{
  "id": "gamenext_component_uuid_<唯一UUID>",  // 每关卡内唯一，跨关卡也不能重复
  "name": "组件名称",
  "base": "MSpine 或 MSprite",
  "edit": {
    "lock": true/false,
    "curState": "state key",
    "baseNotChange": false
  },
  "components": {
    "source": { "MSpine": {"h": ..., "w": ...} },
    "lockState": { "state": false, "componentId": "", "componentName": "", "componentState": "" },
    "judgeRules": { "forbidIfCorrect": false, "inAnswerPool": true/false },
    "webEditorCustomInfo": {
      "chain": "...",              // ⚠️ 见下表，不能为空（对有逻辑的组件）
      "isAnswerComponent": true/false,
      "isJudgeComponent": true/false
    }
  },
  "states": [ ... ],
  "event": { ... }
}
```

### webEditorCustomInfo.chain 填写规则

| 组件 | chain | isAnswer | isJudge |
|---|---|---|---|
| 背景/桥墩/砖块掉落/小鹿开车等 | `""` | false | false |
| 点击选择 | `AloneClickChoice/LComponent/MComponent/cc.Component/cc.Object` | true | true |
| 拖拽放置区N | `LDragPlace/LComponent/MComponent/cc.Component/cc.Object` | true | true |
| 选项N | `MDraggable/MComponent/cc.Component/cc.Object` | true | false |

## 各组件 states 规范

### 砖块掉落（3个 state）

| state key | label | active | source | jump |
|---|---|---|---|---|
| `default` | 默认 | hide | MSpine（URL） | 无跳转 |
| `custom-1729584275015` | 错误反馈 | show | MSpine + MAudio | spinePlayFinish → default |
| `custom-1729584342477` | 错误预显示 | hide | MSpine（+ MAudio 可空） | countdown 1s → 错误反馈 |

砖块掉落 event：
```json
{
  "event": "wrong",
  "action": "changeState",
  "params": { "state": "custom-1729584342477" },
  "source": "gamenext_component_uuid_<对应拖拽放置区N的UUID>",
  "trigger": "all_state"
}
```
**注意：砖块掉落N 的 source 必须指向同关卡的拖拽放置区N（按序号对应）。**

### 拖拽放置区（4个 state）

| state key | label | source |
|---|---|---|
| `default` | 默认 | MSprite |
| `adsorb` | 吸附中 | MSprite |
| `adsorbed` | 已吸附 | MSprite |
| `wrong` | 错误 | MSprite |

### 选项/MDraggable（3个 state）

| state key | label | source |
|---|---|---|
| `default` | 默认 | MLabel + MSprite |
| `placed` | 已放置 | MLabel + MSprite |
| `dragging` | 拖拽中 | MLabel + MSprite |

### 点击选择/AloneClickChoice（4个 state）

| state key | label | source |
|---|---|---|
| `default` | 默认 | MLabel + MSpine |
| `custom-...` | 播放 | MAudio + MLabel + MSpine |
| `level_correct` | 全局正确 | MAudio + MLabel + MSpine |
| `custom-...` | 等待时间 | MAudio + MLabel + MSpine |

点击选择 event：
```json
{
  "event": "click",
  "action": "playAudio",
  "source": "<点击选择自身 id>",
  "trigger": "all_state"
}
```

## UUID 规范

- 格式：`gamenext_component_uuid_<标准UUID>`
- **每个关卡内所有组件 ID 必须唯一**
- **跨关卡也不能重复**（历史问题：Level 2/3/4 曾出现大规模 UUID 重复）
- 砖块掉落和拖拽放置区的关联通过 event.source UUID 建立，必须严格对应

## 常见报错及原因

| 报错 | 原因 | 修复 |
|---|---|---|
| `Cannot read properties of null (reading 'getComponent')` | 顶层 `component_id` 为空字符串 | 按组件类型填正确的 component_id |
| 同上（selectComponent） | chain/component_id 同时缺失 | 补全 webEditorCustomInfo.chain + 顶层 component_id |
| 编辑器属性面板 Spine/Audio 显示空白 | 正常现象（lock=true 的组件只显示模板槽，实际资源在 states 里） | 无需修复 |
| 砖块掉落动画不触发 | event.wrong.source 指向不存在的 UUID | 改为指向同关卡对应拖拽放置区的 id |

## 选项排布规范（2026-04-30 补充）

### 选项尺寸
| 题型 | 选项 w | 选项 h |
|---|---|---|
| 有句号（3~4字）| 286 | 129 |
| 无句号（较长词组）| 379 | 129 |
| 默认/短词 | 166 | 129 |

### 选项 x 坐标计算（统一公式）
```
interval = opt_w + 20   # 20px 间距
total_span = (n-1) * interval
start_x = -total_span / 2
x[i] = start_x + i * interval
```
- 选项必须以屏幕中心（x=0）对称排布
- **禁止**直接使用拖拽放置区的 x 坐标（会导致0间距重叠）

### 节点尺寸（动态计算公式）
节点目的是遮挡所有选项，尺寸必须随选项布局动态计算：
```
left_edge  = min_x - opt_w / 2
right_edge = max_x + opt_w / 2
节点 w = (right_edge - left_edge) + 40   # 左右各 20px padding
节点 h = opt_h + 40                       # 上下各 20px padding
节点 x = (min_x + max_x) / 2
节点 y = opt_y - 10
```

## 转场动效（transition）规范

每个关卡的 `transition` 字段必须完整填写，不能留空壳：

```json
{
  "component_id": "c0d68b01-9045-11ef-b825-e6dfda1bc362",
  "component_name": "通用转场2",
  "component_url": "https://courseware-maker-1252161091.cos.ap-beijing.myqcloud.com/assetboundle/Transition_2/0.0.2/3X/Transition_2.zip",
  "name": "Transition_2",
  "version": "0.0.2",
  "component_data": {
    "base": "MSprite",
    "component_id": "c0d68b01-9045-11ef-b825-e6dfda1bc362",
    "components": {
      "judgeRules": {"forbidIfCorrect": false, "inAnswerPool": true},
      "lockState": {"componentId": "", "componentName": "", "componentState": "", "state": false},
      "source": {"MSprite": 1},
      "tools": {}
    },
    "custom": [],
    "edit": {"baseNotChange": false, "curState": "default", "lock": true},
    "edit_description": "",
    "event": {"dispatchFunList": [], "eventMap": [], "value": []},
    "id": "gamenext_component_uuid_aa949594-e9c9-4727-86a4-2bc0b38a348c",
    "name": "通用转场2",
    "state_group": [],
    "states": [],
    "zIndex": 0
  }
}
```
- 所有关卡可共用同一个 transition id（`aa949594-...`）
- 空壳写法（`component_id: ""`, `name: ""`）不生效，关卡切换无动效

## CDP 保存脚本注意事项（save_game_config_via_cdp.js）

### 已修复问题（2026-04-30）
服务端接口返回的 `detail.result.components` 字段约 **32MB**（组件库模板），如果 PUT 时原样带回，总 payload 超过服务端 **16MB** 限制，导致：
```
服务端错误, 获取数据失败: Unterminated string in JSON at position 16777215
```

**修复方式**：PUT 前排除 `components` 字段：
```js
const { components: _omitComponents, ...detailMeta } = detail.result;
const payload = { ...detailMeta, configuration: innerConfig };
```

此修复已写入脚本，后续无需关注。
