# 开心游乐园 — 动态生成工作流

> 基于 33 个线上配置分析，归纳固定内容与可变内容。  
> 对标过桥大冒险的分析方式，用于实现动态生成脚本。

---

## 一、游戏结构概览

- 每局若干关卡（常见 4～6 关）
- 每关：句子中部分词**已知显示**，部分词需要**拖拽填入空槽**
- 配有**场景图**（情景插画），**选项**可拖入槽中，每个选项有放置音效

```
[场景图(节点)] + [已知条件词1] [拖槽1] [已知条件词2] [拖槽2] ... [句号]
                              ↑ 选项区 ↑（若干个可拖动选项）
```

---

## 二、固定内容（生成时从参考配置 deepcopy，不需修改）

| 组件名 | 类型 | 说明 |
|--------|------|------|
| `背景` | MSprite | 场景底色背景（同系列固定） |
| `补背景` | MSprite | 辅助背景贴图 |
| `节点_10` | MSprite | 装饰节点，固定图片 |
| `关卡数组件` | MSprite + MRichText | 关卡计数UI组件 |
| `小鹿动效` | MSpine | 小鹿常态动效（全局固定 zip） |
| `小鹿动效结算正确` | MSpine | 结算正确动效 |
| `题干喇叭` / `题干喇叭888` | MSpine（壳固定） | Spine 外壳固定；**音频嵌入内部事件，不是 MAudio，无法外部修改** |
| `放置区X错误配合`（X=1~6） | MSpine | 错误反馈 spine，**全局唯一固定 URL** |
| `已知条件左1` | MSprite | 句子框架/背景图片（纯装饰，无文字） |
| `已知条件左5` | MLabel+MSprite | 句号组件，文字固定为 `'.'`（或 `'?'`，见可变部分） |
| `物理遮盖` | — | 遮盖层，固定 |
| `题目配图` | MSprite | 仅少数系列有，且固定（同系列内不变） |

> **注意**：`题干喇叭` 的整句音频通过 Spine 内嵌事件触发，外部无法单独替换 MAudio，需要整体替换 Spine 组件。

## 三、可变内容（每关需要输入的数据）

### 3.1 场景图（节点）

| 字段 | 说明 |
|------|------|
| `节点.states[默认].source.MSprite.value` | 每关的情景插画 URL，**每关不同** |

### 3.2 已知条件词（句子中固定显示的词）

命名规律：`已知条件左`、`已知条件左_21`、`已知条件左2`、`已知条件左3`... 数量因关卡而异。

| 字段 | 说明 |
|------|------|
| `已知条件左X.states[默认].source.MLabel.value` | 显示文字（单词/短语/标点） |
| `已知条件左X.states[默认].source.MSprite.value` | 词卡背景图（同系列固定，只需复用参考配置的 URL） |

### 3.3 拖拽放置区（空槽）

命名规律：`拖拽放置区1`～`拖拽放置区6`，每关槽数量 1～6 不等。

| State | 字段 | 说明 |
|-------|------|------|
| `默认` | MSprite | 空槽背景图（同系列固定） |
| `可吸附` | MSprite | 拖拽悬停时高亮图 |
| `已放置` | MSprite | 已填入时状态图 |
| `错误` | MSprite | 错误时状态图 |

> 拖拽放置区的图片 **同一游戏系列内固定**，直接从参考配置复制即可。

### 3.4 选项（MDraggable）

命名规律：`选项1`～`选项6`，每关选项数 = 槽数 + 1～2 个干扰项。

| State | 字段 | 说明 |
|-------|------|------|
| `默认` | `MLabel.value` | 选项文字 |
| `默认` | `MSprite.value` | 选项词卡背景（同系列固定） |
| `放置` | `MAudio.value` | **放置成功时播放的单词音频 URL（每个词不同）** |
| `放置` | `MLabel.value` | 同默认（保持一致） |
| `放置` | `MSprite.value` | 同默认（保持一致） |

### 3.5 句末标点

`已知条件左5`（或类似命名）的 `MLabel.value`：
- 陈述句：`.`
- 疑问句：`?`
- 感叹句：`！`

## 四、输入数据格式（LEVELS）

每关需要提供以下数据：

```python
LEVELS = [
    {
        "scene_img": "https://.../.../xxxx.png",      # 场景图 URL
        "known": [                                      # 已知条件词列表（按左→右顺序）
            {"text": "He"},
            {"text": "doesn't"},
            {"text": " eggs"},                          # 注意前置空格
        ],
        "slots": 2,                                    # 拖拽空槽数量
        "options": [                                   # 选项列表（含正确+干扰）
            {"text": "like",  "audio": "https://.../like.mp3"},
            {"text": "likes", "audio": "https://.../likes.mp3"},
            {"text": "He",    "audio": "https://.../he.mp3"},
            {"text": "She",   "audio": "https://.../she.mp3"},
        ],
        "correct": ["likes", "He"],                    # 正确答案（按槽顺序）
        "punct": ".",                                  # 句末标点
    },
    # ...更多关卡
]
```

| 内容 | 固定？ | 备注 |
|------|--------|------|
| 背景/补背景 | ✅ 固定 | deepcopy 参考配置 |
| 小鹿动效（2种） | ✅ 固定 | deepcopy |
| 放置区错误配合（所有） | ✅ 固定 | deepcopy，全局唯一 URL |
| 关卡数组件 | ✅ 固定 | deepcopy |
| 题干喇叭（Spine 壳） | ✅ 固定 | deepcopy |
| 词卡背景图（选项/已知条件） | ✅ 固定（同系列） | deepcopy |
| 拖拽槽的 3 态图 | ✅ 固定（同系列） | deepcopy |
| 场景图 | 🔄 每关不同 | 需传入 URL |
| 已知条件词文字 | 🔄 每关不同 | 文字标签 |
| 选项文字 | 🔄 每关不同 | 文字标签 |
| 选项音频 | 🔄 每词不同 | 放置 state MAudio |
| 槽数 / 选项数 | 🔄 每关不同 | 动态增减组件 |
| 句末标点 | 🔄 每关不同 | `.` / `?` / `！` |

## 六、关键注意事项

1. **题干音频无法外部替换**：喇叭组件使用 Spine 内嵌事件，整个 Spine zip 必须含有正确的句子音频。如需替换，需整体替换 Spine 资源。
2. **选项顺序随机**：平台展示时不保证顺序，但配置中的顺序尽量打乱（正确答案不要总是选项1）。
3. **同系列词卡图片可复用**：同一游戏系列（如 starter/level1/level2）的选项背景图、槽图、已知条件背景图是固定的，可以从任意同系列参考配置中提取。
4. **已知条件命名后缀**：后缀数字（`_21`/`_3`等）是编辑器自动生成的，生成时保持与参考配置一致即可。
