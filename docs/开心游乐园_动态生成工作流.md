# 开心游乐园专项生成与校验规则

> 整理自 2026-05-11 实战 + 33 个线上配置分析。
> 核心思路：为「复制替换型」游戏——从参考配置 deepcopy 骨架，只替换可变内容（已知条件文字、选项文字、正确答案、选项音频；场景图仅在明确要求时覆盖）。意图判定、素材同名确认、任务清单、新建游戏、导入保存、回读比对、预览和发布都由主流程 `workflow/` 控制。

---

## 一、主流程挂载点

```text
题目数据 JSON（已知词、选项、正确项、选项音频；可选场景图覆盖）
    ↓ python3 scripts/generate_kaixin_shu2.py --input <题目.json> --template <参考配置.json> --output <输出.config.json>
输出配置 JSON + build-meta
    ↓ 后续导入保存、回读、预览、发布由主流程执行
```

## 二、生成脚本

**路径**：`scripts/generate_kaixin_shu2.py`
**参考配置**：`output/kaixin_refs/57cd2b7b-4b73-11f1-b0f5-e648d636fd2c.json`（国际level2开心游乐园暑2，1槽×4关）

### 运行

```bash
cd CoursewareMaker_Automation_Toolkit
python3 scripts/generate_kaixin_shu2.py
```

输出：`output/kaixin_shu2/kaixin_shu2.json`

正式任务必须使用动态输入：

```bash
python3 scripts/generate_kaixin_shu2.py \
  --input data/amusement_park_questions.json \
  --template output/kaixin_refs/57cd2b7b-4b73-11f1-b0f5-e648d636fd2c.json \
  --output output/kaixin_shu2/<游戏名>.json \
  --meta output/kaixin_shu2/<游戏名>.build-meta.json
```

### 生成策略

**复制替换型**（非动态布局）：

1. 从参考配置 deepcopy 固定组件（背景、小鹿动效、错误配合、词卡背景图等）
2. 替换关卡可变内容：已知条件文字、选项文字、正确答案、选项音频；只有输入显式提供 `scene_img` / `scene_img_override` 时才覆盖场景图
3. **不重算坐标**——所有组件位置、尺寸沿用参考配置的原值

> ⚠️ 这意味着参考配置选定后，槽数、选项数、已知条件词数都跟着参考模板走。
> 如需不同槽数/选项数，选择不同槽数的参考配置作为模板。

## 三、游戏结构概览

**⚠️ 渲染层级规则：`components[]` 数组顺序决定渲染层级，越靠后越在上层。zIndex 字段仅作编辑器显示参考，不影响实际渲染。**

```
[背景]                         ← 第0层 最底层
[小鹿动效]                     ← 第1层
[节点(场景图)]                 ← 第2层
[关卡数组件]                   ← 第3层
[已知条件左X] ...             ← 句子固定词
[题干喇叭/喇叭888]             ← Spine内嵌音频
[句号]
[放置区X错误配合] ...          ← 1:1匹配每个槽
[拖拽放置区X] ...             ← 槽
[选项X] [选项X] ...            ← 在拖槽之后 = 渲染在槽之上（可交互拖拽）
[小鹿动效结算正确]
[物理遮盖]                     ← 最后 = 最上层，盖住所有内容
```

### 组件类型说明

| 组件名 | 类型 | 说明 |
|--------|------|------|
| 背景 | MSprite | 固定底色（2048×1152） |
| 小鹿动效 | MSpine | 常态 spine，固定资源 |
| 小鹿动效结算正确 | MSpine | 正确反馈 spine，固定资源 |
| 节点_10 | MSprite | 装饰层，固定图像 |
| 节点（场景图） | MSprite | 默认继承参考模板；只有输入显式提供时才覆盖 |
| 已知条件左X | MLabel + MSprite | 句子中固定显示的词，**每关替换文字** |
| 拖拽放置区X | LDragPlace | 空槽，1~6 个不等，**每关替换答案 tag** |
| 放置区X错误配合 | MSpine | 错误反馈动画，固定资源，1:1 匹配槽 |
| 关卡数组件 | LevelNumber bundle | 关卡计数 UI，固定资源 |
| 选项X | MDraggable | 拖拽词卡，**每关替换文字 + 音频** |
| 题干喇叭 | MSpine | 整句播放，**Spine 内嵌音频无法外部替换** |
| 句号 | MLabel | 句末标点，固定 `.` 或 `?` |
| 物理遮盖 | MSprite | 顶部遮盖层，固定资源 |

## 四、题目数据格式

每个关卡定义如下：

```json
{
  "levels": [
    {
      "fixed_text": "stickers",
      "options": ["collect", "draw", "watch"],
      "correct": "collect",
      "option_audio_urls": ["https://.../collect.mp3", "https://.../draw.mp3", "https://.../watch.mp3"],
      "punct": "."
    }
  ]
}
```

正式输入中 `fixed_text/options/correct/option_audio_urls` 都是题目相关信息，必须来自 `--input`。`scene_img` 不是必填；开心游乐园和过桥大冒险一样先继承自己的参考资源模板，只有用户明确要求替换情景插画时才提供 `scene_img` 或 `scene_img_override`。脚本内 `LEVELS` 和 `AUDIO_PLACEHOLDER` 只保留为旧样例兼容；正式任务若缺少选项音频会阻断。

### 多槽版本（如有扩展需求）

```python
{
    'scene_img':  'https://.../scene.png',
    'known': [                                    # 已知条件词（按左→右顺序）
        {'text': 'He'},
        {'text': "doesn't"},
        {'text': ' likes'},
    ],
    'slots': 2,                                  # 拖拽槽数
    'options': [                                 # 选项列表（含正确+干扰）
        {'text': 'like',  'audio': 'https://.../like.mp3'},
        {'text': 'likes', 'audio': 'https://.../likes.mp3'},
    ],
    'correct': ['likes'],                        # 正确答案（按槽顺序）
    'punct': '.',
}
```

| 关卡数组件 | ✅ 固定 | deepcopy |

## 六、选项构建规则

从参考选项组件 deepcopy 后替换：

```python
def make_option_comp(ref_comp, text, audio_url):
    c = copy.deepcopy(ref_comp)
    c['component_data']['id'] = str(uuid.uuid4())
    # 替换所有状态的文字
    for label in ['默认', '放置', '当前组件错误', '拖拽中']:
        st = get_state(c['component_data'], label)
        if st:
            set_mlabel(st, text)
    # 拖拽中 state → 播放放置音频
    st = get_state(c['component_data'], '拖拽中')
    if st:
        set_maudio(st, audio_url)
    return c
```

**关键字段：**
- `MDraggable.tag` = 选项文字（用于匹配放置区 answer）
- `MLabel.value` = 选项显示文字
- 拖拽中 state 的 `MAudio.value` = 放置音效

## 七、坐标与尺寸（来自参考配置，不做动态计算）

开心游乐园不重算坐标，所有组件直接从参考配置继承。以 edc8d540（starter 春12，2槽）为例：

| 组件 | 默认 x | y | w | h |
|------|--------|------|------|------|
| 背景 | 0 | 0 | 2048 | 1152 |
| 小鹿动效 | -8 | 8 | 2071 | 1152 |
| 节点(场景图) | 311 | 331 | 501 | 300 |
| 关卡数组件 | 745 | 485 | 322 | 139 |
| 句号 | 655 | -84 | 50 | 170 |
| 放置区 Y | 约 -85 | (各槽 x 不同) | 247 | 165 |
| 已知条件 Y | 约 -84 | (各词 x 不同) | 250 | 170 |
| 选项区 Y | -392 | (各选项 x 不同) | 250 | 170 |

> **不同系列（starter/level1/level2）的参考配置可能有不同坐标和组件命名**（如 `已知条件左5` vs `已知条件3`）。
> 生成脚本需针对不同参考配置适配组件名称查找逻辑。

## 八、主流程保存边界

本文只产出配置 JSON。保存到已有 `game_id`、新建游戏、回读比对、预览和发布必须回到主流程，由 `workflow_planner.py` 和 `workflow_guard.py` 决定是否允许执行。

保存语义统一由主流程脚本负责：`GET 完整游戏元信息 -> 只替换 configuration -> 保留或补齐 components=[] -> PUT /beibo/game/config/game`。`configuration` 必须是对象，不能双重编码成字符串。

## 九、顶层 JSON 必要字段

```json
{
  "additional":  { ...从参考配置 deepcopy... },
  "common":      { ...从参考配置 deepcopy... },
  "components":  [ ...组件类型注册表，从参考配置 deepcopy... ],
  "game":        [ ...各关卡... ]
}
```

> ⚠️ 开心游乐园**没有 `meta` 字段**（与过桥大冒险不同），但必须有 `additional`、`common`、`components`、`game`。

## 十三、与过桥大冒险的核心区别

| 维度 | 过桥大冒险 | 开心游乐园 |
|------|-----------|-----------|
| 生成方式 | 动态生成型 | 复制替换型 |
| 坐标来源 | 从文字长度实时计算 | 从参考配置 deepcopy，不变 |
| 槽数灵活性 | 同一脚本支持任意槽数 | 换不同槽数的参考配置 |
| 选项宽度 | 统一槽宽（防提示） | 参考配置固定 |
| 字号 | 自适应（readonly>50） | 参考配置固定（90px） |
| 组件构建 | `make_slot()` `make_pillar()` 函数组装 | deepcopy 参考组件后替换字段 |
| 模板依赖 | 只有组件骨架，坐标重新算 | 坐标完全依赖模板 |

## 十四、调试备忘

| 问题 | 原因 | 解法 |
|------|------|------|
| 关卡数组件不显示 | 缺少 `components` 注册表 | 从参考配置 deepcopy top-level `components` 数组 |
| 选项无法拖入槽 | MDraggable.tag 与 LDragPlace.itemList 不匹配 | 确认 tag 值与 answer 值一致 |
| 选项被背景遮挡 | 选项在 `components[]` 中位置太靠前 | 选项必须放在数组最后，渲染在最上层才能交互 |
| 放置后无音效 | 拖拽中 state 缺少 MAudio | 替换 `拖拽中` state 的 `MAudio.value` |
| 已知条件词换行/溢出 | 文字过长超过词卡宽度 | 缩短文字或选更宽的参考配置 |
| 题干喇叭不播放 | Spine 内嵌音频，无法外部替换 | 需整体替换 Spine 组件或上传新 Spine zip |
| 组件命名不匹配（`已知条件左5` vs `已知条件3`） | 不同系列的命名规则不同 | 用 `name_contains` 模糊匹配而非精确匹配 |
| PUT 返回 404 | API 路径或 method 错误 | 确认 `PUT /beibo/game/config/game` |
| Token 过期 | JWT exp 过期 | 重新从浏览器 `localStorage["GAMEMAKER_TOKEN"]` 获取 |
